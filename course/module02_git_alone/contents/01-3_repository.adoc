= 저장소(Repository)

저장소(Repository, 리포지토리)는 **프로젝트의 모든 파일과 그 파일들의 변경 이력(history)이 영구적으로 저장되는 데이터베이스**입니다.

작업 디렉토리가 '현재 작업중인 책상'이고, 스테이징 영역이 '포장된 박스'라면, 저장소는 **'완성된 기록이 보관된 문서고'**입니다.

== 저장소의 종류: 로컬 vs. 원격

Git의 가장 큰 특징은 분산 버전 관리 시스템이라는 점입니다. 따라서 저장소는 크게 두 군데로 나뉩니다.

1. 로컬 저장소(Local Repository)
* 위치: 내 컴퓨터의 프로젝트 폴더 안에 숨겨진 `.git` 폴더
* 역할: 내가 작업흔 모든 버전(커밋)이 저장되는 개인 금고
* 특징: 네트워크 연결 없이도 과거 내역 조회, 버전 되돌리기, 새로운 버전 생성(커밋)이 가능합니다. 내 컴퓨터에 저장되므로, 작업이 유실될 수 있습니다.
2. 원격 저장소(Remote Repoistory)
* 위치: GitHub, GitLab, Azure DevOps 같은 인터넷상의 서버입니다.
* 역할: 협업을 위한 중앙 거점이자 백업 공간입니다.
* 특징: 여러 개발자가 자신의 로컬 저장소를 이곳으로 밀어 올리고(Push), 남의 작업 내용을 가져옵니다(Pull).

== 저장되는 데이터

저장소(특히 `.git` 폴더)안에는 단순한 파일 복사본이 아닌, Git이 관리하는 특수한 데이터들이 들어있습니다.

1. **Commit** +
특정 시점의 파일 상태를 사진 찍듯이 기록한 스냅샷입니다. 누가, 언제, 무엇을 변경 했는지에 대한 정보가 담겨 있습니다.
2. **Branch 및 Tag**: +
특정 이름을 가리키는 포인터(이름표)입니다.
3. **HEAD** +
"현재 내가 보고 있는 버전이 어디인가"를 가리키는 포인터입니다.
4. **Config** +
프로젝트의 설정 정보(사용자 이름, 이메일, 원격 저장소 주소 등)가 들어있습니다.
