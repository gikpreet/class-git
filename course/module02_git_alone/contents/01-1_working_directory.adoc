= 작업 디렉토리(Working Directory)
:sectnums:


* 사용자의 작업 공간
* 

---

Git 작업 디렉토리는 사용자가 실제로 눈으로 보고, 파일을 작성하고, 수정하는 사용자의 작업 공간을 의미합니다. 간단히 말해, 현재 컴퓨터에 들어 있는 실제 파일들이 바로 작업 디렉토리입니다.

== 작업 디렉토리의 개념적 역할

작업 디렉토리는 작업대(Workbench)와 같습니다. 창고(저장소)에서 물건을 가져와 작업대 위에 올려주고 자르고, 붙이, 저장하는 공갼입니다.

* 가시성: 사용자가 윈도우 탐색기나 Visual Studio Code 탐색기에서 직접 볼 수 있는 파일들입니다.
* 휘발성: 이곳에서 내용을 수정하더라도, Git에게 저장하라고 명령(Commit)하지 않으면 과거의 기록으로 남지 않습니다. 실수를 해도 커밋하기 전이라면 언제든 되돌릴 수 있습니다.
* 독립성: 작업 디렉토리는 .git 폴더(숨김 폴더)를 제외한 나머지 모든 영역을 말합니다. .git 폴더는 변경 이력을 담은 데이터베이스이고, 작업 디렉토리는 그 데이터베이스에서 꺼내욘 특정 시점의 스냅샷입니다.

== 작업 디렉토리 안에서의 파일 상태

작업 디렉토리에 있는 파일들을 Git의 관점에서 크게 두 가지 상태로 나뉩니다. 이 상태 변화를 이해하는 것이 중요합니다.

1. **Tracked(관리 대상)**: Git이 미리 알고 있고, 이전에 커밋된 적이 있는 파일들입니다. 이 파일들은 세 가지 상태로 나뉩니다.
* Unmodified(수정 없음): 마지막 커밋 이후 내용이 변하지 않은 상태입니다.
* Modified(수정됨): 마지막 커밋 이후 내용이 변경되었지만, 아직 스테이징 영역에 올라가지 않는 상태입니다.
* Staged(스테이징됨): 수정된 파일을 곧 커밋하겠다고 표시해 둔 상태입니다.
2. **Untracked(관리 대상 아님)**: 파일을 새로 생성했지만, 아직 한 번도 Git에 커밋하지 않은 파일입니다. Git은 사용자가 이 파일을 명시적으로 추가(Add) 하기 전까지는 이 파일의 변화를 추적하지 않습니다.
