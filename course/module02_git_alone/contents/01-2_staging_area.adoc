= 스테이징 영역(Staging Area)

스테이징 영역은 **커밋(저장)하기 직전의 파일들이 대기하는 임시 공간입니다.

작업 디렉토리가 '자유로운 작업 공간'이라면, 스테이징 영역은 '저장할 것들만 골라 놓은 선별 공간'입니다. Git 내부적으로는 **인덱스(Index)**라고도 부릅니다.

== 스테이징 영역이 필요한 이유

스테이징 영역이 필요한 이유는 **'원하는 만큼만 쪼개서 저장하기 위해서'** 입니다.

만약 개발자가 아래와 같은 두 가지 작업을 동시에 작업한 경우,

1. 로그인 버그 수정(파일 A 수정)
2. 메인 페이지 디자인(파일 B, C 수정)

이때 스테이징 영역이 없다면, '버그 수정'과 '디자인 변경'이라는 전혀 다른 두 가지 내용이 하나의 커밋(버전)으로 저장됩니다. 스테이징 영역은 이 두 작업을 작업의 이력으로 분리해서 관리할 수 있도록 합니다.

1. 파일 A만 스테이징 영역에 올리고 '로그인 버그 수정'이라고 커밋합니다.
2. 파일 B, C를 스테이징 영역에 올리고 '메인 페이지 디자인 변경'이라고 커밋합니다.

스테이징 영역은 이런 방식으로 **작업의 이력을 논리적으로 깔끔하게 분리**해서 관리할 수 있습니다.
