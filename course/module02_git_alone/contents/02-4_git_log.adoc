= git log 명령

`git log` 는 프로젝트의 history를 펼쳐보는 명령어입니다. 

지금 까지 저장된 모든 커밋 내역을 최신순으로 보여줍니다. 누가, 언제, 무엇을, 왜 변경했는지 추적할 때 사용됩니다.

== 기본 실행 방법

기본 명령 사용 방법은 아래와 같습니다.

----
git log
----

출력 결과는 아래와 같습니다.

----
commit ca82a6dff817ec66f44342007202690a93763949 (HEAD -> main)
Author: Hong Gildong <hong@example.com>
Date:   Mon Dec 9 14:00:00 2025 +0900

    로그인 기능 추가

commit 8a2b3c...
Author: ...
...
----

1. commit (해시값): ca82a6... 같은 긴 문자열은 해당 커밋의 **고유 ID(주민등록번호)**입니다. 나중에 특정 시점으로 되돌아갈 때 이 번호를 사용합니다.
2. HEAD -> main: 현재 내 위치(HEAD)가 main 브랜치의 이 커밋을 보고 있다는 뜻입니다.
3. Author/Date: 누가, 언제 수정했는지 기록입니다.
4. Message: git commit -m으로 작성했던 설명입니다.

== 옵션

옵션 없이 사용한 git log 명령은 길게 표시되어 가독성이 떨어질 때가 많습니다. 아래 옵션들을 사용할 수 있습니다.

=== `--oneline` 옵션

가장 많이 쓰는 옵션이며, 해시값의 앞부분(7자리)과 제목만 깔끔하게 보여줍니다.

----
$ git log --oneline
2b1f84f (HEAD -> main, origin/main, origin/HEAD) Initial commit
----

=== `--graph` 옵션

브랜치가 갈리지고 합쳐지는 모양을 아스키 아트(ASCII Art) 선으로 보여줍니다.

----
git log --oneline --graph --all
----

* `--all`: 현재 브랜치뿐만 아니라 모든 브랜치의 history를 보여줍니다.
* 이 옵션 조합은 **"A Dog(All, Decorate, Oneline, Graph)"**라고 외우면 편합니다.

=== `-n` 옵션

-n 옵션은 원하는 개수의 history 목록을 출력합니다.

----
git log -3
----

=== `'파일 이름'` 옵션

툭정 파일이 언제 어떻게 바뀌었는지 추적할 때 사용합니다.

----
git log index.html
----

`-p` 옵션을 사용하면 파일 내부의 어떤 코드가 바뀌었는지 구체적인 내용을 보여줍니다.

----
git log -p index.html
----

=== `--author` 옵션

특정 팀원이 작업한 내역만 찾을 때 사용합니다.

----
git log --author="anthrix"
----

=== `--grep` 옵션

커밋 메시지의 내용으로 로그를 검색합니다. 

----
git log --grep="버그"
----
